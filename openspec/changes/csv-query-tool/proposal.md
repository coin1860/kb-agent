## Why

Currently, querying CSV files relies on basic loading and lacks a robust, safe way to execute complex data retrieval operations. Implementing a specific CSV querying tool that utilizes LLM-generated Pandas queries ensures sufficient retrieval power without exposing the system to security injection risks.

## What Changes

- Implement priority loading for CSV files based on configured paths (`archive/` then `source/`).
- Add an LLM interaction flow to translate user intents into safe Pandas `.query()` strings.
- Create a dedicated python tool (`csv_qa_tool.py`) with memory caching to handle requests.
- Update the Agent Router prompting to properly direct CSV file queries to this new tool.
- Clear pandas memory cache when chat history is cleared.

## Capabilities

### New Capabilities

- `csv-query`: Direct querying mechanism for CSV files using safe Pandas `.query()` statements generated by an LLM with structured output.

### Modified Capabilities

- `routing-engine`: Updated rules to accurately route `.csv` file related queries directly to the new `csv-query` tool.

## Impact

- Introduces a new tool (`csv_qa_tool.py`) with Pandas dependency logic.
- Modifies `kb_agent/agent/nodes.py` to update the decomposition and planning prompts.
- Modifies `kb_agent/tui.py` and `kb_agent/agent/tools.py` for cache lifecycle management.
